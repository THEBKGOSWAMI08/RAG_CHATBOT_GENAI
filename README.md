# RAG-based Q&A Chatbot

A Retrieval-Augmented Generation (RAG) application built with Streamlit that lets you upload documents and ask grounded questions about them. Answers are generated by Groq-hosted LLMs using context retrieved from a local FAISS vector store, with an optional cross-encoder reranker for higher precision.

---

## Features

- **Multi-format ingestion** вЂ” PDF, TXT, and DOCX files via the Streamlit sidebar
- **Local vector search** вЂ” FAISS index persisted on disk (`faiss_db/`)
- **Dense embeddings** вЂ” `all-MiniLM-L6-v2` (Sentence Transformers)
- **Optional reranking** вЂ” `cross-encoder/ms-marco-MiniLM-L-6-v2` for top-k refinement
- **Conversation memory** вЂ” last 10 turns injected into each prompt
- **Groq LLM backend** вЂ” defaults to `llama-3.1-8b-instant` for low-latency generation
- **Reset knowledge base** вЂ” wipe the index and memory in one click

## Requirements

- Python **3.10 вЂ“ 3.12**
- [Poetry](https://python-poetry.org/) for dependency management
- A [Groq API key](https://console.groq.com/)

---

## Setup

### 1. Clone and enter the project

```bash
git clone <your-repo-url>
cd RAG
```

### 2. Install Poetry (if you don't have it)

```powershell
(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | py -
```

### 3. Install dependencies

```bash
poetry install
```

Poetry creates an in-project virtualenv at `.venv/` and installs everything from `poetry.lock`.

### 4. Configure your API key

Create a `.env` file at the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
```

---

## Running the App

Recommended:

```bash
poetry run python run.py
```

The launcher starts Streamlit in headless mode, pre-warms the model cache, and prints a clickable URL only when the app is fully ready вЂ” so the page opens instantly instead of showing a cold-start spinner. Press **Ctrl+C** to stop.

Alternative (direct Streamlit, skips the warm-up):

```bash
poetry run streamlit run app/app.py
```

From the UI:

1. Upload one or more PDF / TXT / DOCX files in the sidebar
2. Click **Ingest Documents**
3. Ask questions in the main panel вЂ” toggle the reranker on/off as needed
4. Click **Reset Knowledge Base** to clear the index and start over

## Configuration

Tunable parameters live inside each module:

| Parameter                | Location                           | Default                                   |
| ------------------------ | ---------------------------------- | ----------------------------------------- |
| Chunk size / overlap     | `src/chunking/chunking.py`         | 500 / 50                                  |
| Embedding model          | `src/embedding/embedder.py`        | `all-MiniLM-L6-v2`                        |
| Retriever top-k          | `src/pipeline/rag_pipeline.py`     | 5                                         |
| Reranker model / top-k   | `src/reranker/reranker.py`         | `ms-marco-MiniLM-L-6-v2` / 3              |
| LLM model / temperature  | `src/llm/llm.py`                   | `llama-3.1-8b-instant` / 0.3              |
| Memory turns             | `src/pipeline/rag_pipeline.py`     | 10                                        |

## Tech Stack

- **Framework:** Streamlit
- **Orchestration:** LangChain (core, community, text-splitters, groq)
- **Vector store:** FAISS (CPU)
- **Embeddings / Reranking:** Sentence Transformers
- **LLM:** Groq (`llama-3.1-8b-instant`)
- **Document parsing:** pypdf, python-docx
- **Env management:** Poetry, python-dotenv
